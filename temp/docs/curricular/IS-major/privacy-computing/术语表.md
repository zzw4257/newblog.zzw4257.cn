## 密码学与安全多方计算术语表

### 安全多方计算 (Secure Multi-Party Computation, MPC)

| 术语 (Term - Chinese) | 英文 (English)                        | 解释 (Explanation)                                                                                                                                                                             |
| :-------------------- | :------------------------------------ | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 安全多方计算          | Secure Multi-Party Computation (MPC)  | 一种密码学分支，允许多个参与方在不泄露各自私有输入的情况下，共同计算一个约定函数的结果。计算过程需保证**正确性**（输出结果正确）和**隐私性**（除计算结果外，不泄露任何额外信息）。 |
| 隐私计算              | Privacy Computing                     | 一个更广泛的领域，包含 `MPC` 在内的多种技术和方法，旨在数据处理和使用过程中保护数据隐私。                                                                                                    |
| 通用解决方案          | Universal Solution / General Protocol | 指能够安全计算*任何*函数的 `MPC` 协议，通常通过将函数表示为算术电路或布尔电路来实现。                                                                                                      |

### 基础数学概念 (Foundational Mathematics)

| 术语 (Term - Chinese) | 英文 (English)            | 解释 (Explanation)                                                                                                                                                                                                                                                                                                                                   |
| :-------------------- | :------------------------ | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 数论基础              | Number Theory Foundations | 构成许多密码协议基础的数论基本概念。                                                                                                                                                                                                                                                                                                                 |
| 模                    | Modulus                   | 在模运算中（例如$a \pmod n$），模 $n$ 是指用一个整数去除另一个整数以求余数时的除数。                                                                                                                                                                                                                                                             |
| 群 (G)                | Group                     | 一个集合$G$ 配备一个二元运算（例如 $\cdot$），满足四个公理：闭包性（对所有 $a, b \in G$，有 $a \cdot b \in G$）、结合性（$(a \cdot b) \cdot c = a \cdot (b \cdot c)$）、单位元（存在 $e \in G$ 使得 $a \cdot e = e \cdot a = a$）和逆元（对每个 $a \in G$，存在 $a^{-1} \in G$ 使得 $a \cdot a^{-1} = a^{-1} \cdot a = e$）。    |
| 环                    | Ring                      | 一种代数结构$(R, +, \cdot)$，具有加法和乘法两种二元运算，使得 $(R, +)$ 是一个阿贝尔群，乘法具有结合性，且乘法对加法满足分配律。                                                                                                                                                                                                                  |
| 域 (F)                | Field                     | 一种环，其中加法和乘法均满足交换律，存在乘法单位元，并且每个非零元素都有乘法逆元。                                                                                                                                                                                                                                                                   |
| 有限域 (Fp 或 Zp)     | Finite Field              | 包含有限个元素的域。常见的例子是整数模素数$p$ 的集合，记为 $\mathbb{Z}_p = \{0, 1, \dots, p-1\}$ 或 $\mathbb{F}_p$。                                                                                                                                                                                                                           |
| 循环群                | Cyclic Group              | 可以由单个元素（称为生成元）生成的群。群中的所有元素都是该生成元的幂（或在加法表示法中的倍数）。                                                                                                                                                                                                                                                     |
| 生成元 (g)            | Generator                 | 循环群$G$ 中的一个元素 $g$，使得 $G$ 中的每个元素都可以表示为某个整数 $k$ 的 $g^k$（对于乘法群）或 $k \cdot g$（对于加法群）。                                                                                                                                                                                                           |
| 拉格朗日插值法        | Lagrange Interpolation    | 一种寻找通过给定一组不同点的最低可能次数的唯一多项式的方法。对于$l+1$ 个点 $(\alpha_1, q(\alpha_1)), \dots, (\alpha_{l+1}, q(\alpha_{l+1}))$，次数最多为 $l$ 的多项式 $q(x)$ 由 $q(x) = \sum_{i=1}^{l+1} q(\alpha_i) \delta_i(x)$ 给出，其中 $\delta_i(x) = \prod_{1 \le j \le l+1, j \ne i} \frac{x - \alpha_j}{\alpha_i - \alpha_j}$。 |

### 现代密码学基础 (Modern Cryptography Foundations)

| 术语 (Term - Chinese)    | 英文 (English)                      | 解释 (Explanation)                                                                                                                                                                                                                                                                                                                     |
| :----------------------- | :---------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 可证明安全               | Provable Security                   | 一种严格的密码学方法，其中方案的安全性被形式化定义，基于精确的假设（例如，数学问题的难度），并通过严格的数学论证（通常是规约）来证明。                                                                                                                                                                                                 |
| 规约                     | Reduction                           | 可证明安全中常用的一种证明技巧。它表明如果一个敌手$A$ 能够攻破一个密码协议 $\Pi$，那么可以构造一个算法 $B$（使用 $A$ 作为子程序）来解决一个被假定为困难的底层硬问题。这意味着只要底层假设成立，$\Pi$ 就是安全的。                                                                                                            |
| 可忽略函数 (negl(κ))    | Negligible Function                 | 安全参数$\kappa$ 的一个函数 $\text{negl}(\kappa)$，它比 $\kappa$ 的任何正多项式的倒数更快地趋近于零。形式上，对于任何正多项式 $\text{poly}(\cdot)$，存在一个 $N$，使得对于所有 $\kappa > N$，都有 $\text{negl}(\kappa) < \frac{1}{\text{poly}(\kappa)}$。这些函数用于限制敌手的成功概率。                                |
| 概率多项式时间 (PPT)     | Probabilistic Polynomial Time (PPT) | 一种算法，其运行时间由其输入大小的多项式界定，并且可能使用随机性。密码安全定义中的敌手通常被建模为 `PPT` 算法。                                                                                                                                                                                                                      |
| 计算上不可区分 (X1≈cX2) | Computationally Indistinguishable   | 如果没有 `PPT` 算法（区分器）$D$ 能够以超过可忽略的概率区分从 $X_\kappa$ 和 $Y_\kappa$ 中抽取的样本，则两个概率分布族 $\{X_\kappa\}_{\kappa \in \mathbb{N}}$ 和 $\{Y_\kappa\}_{\kappa \in \mathbb{N}}$ 是计算上不可区分的。即，$|\Pr[D(1^\kappa, X_\kappa)=1] - \Pr[D(1^\kappa, Y_\kappa)=1]| \le \text{negl}(\kappa)$。 |

### 密码学基础原语 (Basic Cryptographic Primitives)

| 术语 (Term - Chinese)                | 英文 (English)                                   | 解释 (Explanation)                                                                                                                                                                                                                                                                                                 |
| :----------------------------------- | :----------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 秘密分享                             | Secret Sharing                                   | 一种将秘密分成多个片段（称为份额）的方法，这些份额分发给多个参与方。单个份额（或少数份额）不泄露关于秘密的任何信息，但足够数量的份额可以组合起来重建原始秘密。                                                                                                                                                     |
| 门限秘密分享                         | Threshold Secret Sharing                         | 一种秘密分享方案，记为$(t,n)$-门限秘密分享，其中秘密被分成 $n$ 个份额。任何 $t+1$（或更多）个份额可以重建秘密，但任何 $t$（或更少）个份额不泄露关于秘密的任何信息。                                                                                                                                        |
| Shamir 秘密分享                      | Shamir Secret Sharing                            | 一种基于有限域上多项式插值的$(t,n)$-门限秘密分享方案。为了分享秘密 $s$，选择一个次数最多为 $t$ 的随机多项式 $q(x)$，使得 $q(0)=s$。份额是 $s_j = q(\alpha_j)$，其中 $\alpha_j$ 是不同的非零元素。                                                                                                    |
| 秘密份额                             | Secret Share                                     | 秘密分享方案中秘密的单个片段。                                                                                                                                                                                                                                                                                     |
| 哈希函数 (H)                         | Hash Function                                    | 将任意大小的输入数据映射到固定大小输出（称为哈希值或摘要）的函数。密码学哈希函数被设计为单向的（抗原像性）、抗第二原像性和抗碰撞性。                                                                                                                                                                               |
| 随机谕示机 (RO)                      | Random Oracle                                    | 一种理想化的数学函数，对每个唯一的查询都以一个真正随机的输出（从其输出域中均匀选择）进行响应，并对相同的查询始终返回相同的随机输出。它是在安全证明中使用的理论模型。                                                                                                                                               |
| 伪随机数生成器 (PRG) (G)             | PseudoRandom Generator (PRG)                     | 一种确定性算法$G$，它以一个短的随机种子 $s$ 作为输入，并输出一个更长的字符串 $G(s)$，该字符串在计算上与相同长度的真正随机字符串不可区分。形式上，对于任何 `PPT` 区分器 $A$， $|\Pr[A(1^\kappa, G(s))=1] - \Pr[A(1^\kappa, r)=1]| \le \text{negl}(\kappa)$，其中 $s$ 是随机种子，$r$ 是随机字符串。 |
| 对称加密                             | Symmetric Encryption                             | 一种加密类型，其中加密和解密使用相同的密钥。它包括三个算法：密钥生成 (`Gen`)、加密 (`Enc_k(m)`) 和解密 (`Dec_k(c)`)。必须满足 $Dec_k(Enc_k(m))=m$。                                                                                                                                                        |
| 消息认证码 (MAC)                     | Message Authentication Code (MAC)                | 用于验证消息并确保其完整性和真实性的一小段信息。它涉及参与方之间共享的密钥。包括算法：密钥生成 (`Gen`)、`MAC` 生成 (`Mac_k(m)`) 和验证 (`Ver_k(m,t)`)。                                                                                                                                                    |
| 信息论一次性消息认证码               | Information-Theoretic One-Time MAC               | 一种 `MAC` 方案，其安全性（不可伪造性）即使对于具有无限计算能力的敌手也成立，但通常密钥只能用于验证单个消息。例如：$Mac_{a,b}(m) = (a \cdot m + b) \pmod p$。                                                                                                                                                  |
| 承诺方案                             | Commitment Scheme                                | 一种密码学协议，允许一方（承诺者）承诺一个选定的值（或陈述），同时对另一方（接收者）隐藏该值，并能在以后揭示承诺的值。它具有两个主要属性：**隐藏性**（接收者在打开之前对值一无所知）和**绑定性**（承诺者在承诺后不能更改值）。算法：(`Gen`、`Com`、`Ver`)。                                      |
| 承诺                                 | Commitment                                       | `Com` 算法输出的值。                                                                                                                                                                                                                                                                                             |
| 隐藏性                               | Hiding                                           | 承诺$c$ 不泄露消息 $m$ 的属性。                                                                                                                                                                                                                                                                                |
| 绑定性                               | Binding                                          | 承诺者找不到$m' \ne m$ 和打开信息 $d,d'$ 使得承诺 $c$ 可以有效地打开为 $m$ 和 $m'$ 的属性。                                                                                                                                                                                                              |
| 茫然传输 (OT)                        | Oblivious Transfer (OT)                          | 发送者和接收者之间的两方协议。                                                                                                                                                                                                                                                                                     |
| 1-out-of-2 OT                        | 1-out-of-2 OT                                    | 发送者有两个消息$(x_0,x_1)$，接收者有一个选择位 $b \in \{0,1\}$。接收者学习到 $x_b$ 但对 $x_{1-b}$ 一无所知。发送者对 $b$ 一无所知。                                                                                                                                                                     |
| 理想功能$F_{OT}$                   | Ideal Functionality$F_{OT}$                    | 一个理想化的可信第三方，它从发送者处获取$(x_0,x_1)$，从接收者处获取 $b$，并私下将 $x_b$ 交给接收者。                                                                                                                                                                                                         |
| Rabin OT                             | Rabin OT                                         | 一种变体，发送者有一个消息$x$，接收者以 $1/2$ 的概率接收到 $x$，或以 $1/2$ 的概率接收到一个虚拟值。发送者不知道接收者是否得到了 $x$。                                                                                                                                                                    |
| 随机 OT                              | Random OT                                        | 发送者和接收者获得随机相关输出：发送者获得$(x_0,x_1)$，接收者获得 $(b,x_b)$，其中 $x_0,x_1$ 是随机消息，$b$ 是随机位。                                                                                                                                                                                     |
| 1-out-of-N OT                        | 1-out-of-N OT                                    | 发送者有$N$ 个消息 $(x_0,\dots,x_{N-1})$，接收者有一个索引 $b \in \{0,\dots,N-1\}$。接收者只学习到 $x_b$。发送者对 $b$ 一无所知。                                                                                                                                                                        |
| k-out-of-N OT                        | k-out-of-N OT                                    | 发送者有$N$ 个消息，接收者有 $k$ 个索引。接收者只学习到 $k$ 个选定的消息。发送者对这 $k$ 个索引一无所知。                                                                                                                                                                                                  |
| 茫然传输扩展 (OT Extension)          | Oblivious Transfer Extension (OT Extension)      | 从少量“基础”`OT` 高效生成大量 `OT` 的技术，通常使用对称密钥密码学。                                                                                                                                                                                                                                          |
| 切分选择 OT (CCOT)                   | Cut-and-Choose OT (CCOT)                         | 一种 `OT` 变体，用于切分选择协议，其中接收者除了获得选定的消息外，还可以检查发送者输入的一个子集的一致性。                                                                                                                                                                                                       |
| 单一选择切分选择 OT (FCCOTS)         | Single-Choice Cut-and-Choose OT (FCCOTS)         | 一种 `CCOT`，其中接收者对所有不在“检查”集中的 `OT` 实例使用相同的选择位。                                                                                                                                                                                                                                    |
| 批处理单一选择切分选择 OT (FCCOTS,B) | Batch Single-Choice Cut-and-Choose OT (FCCOTS,B) | 将单一选择 `CCOT` 扩展到高效处理多组消息（批处理），通常用于电路中的不同输入线，同时使用相同的“检查”索引集。                                                                                                                                                                                                   |
| 零知识证明 (ZKP)                     | Zero-Knowledge Proof (ZKP)                       | 一种协议，其中一方（证明者）可以向另一方（验证者）证明一个陈述是真实的，而不泄露除陈述有效性之外的任何信息。关键属性是**完备性**、**可靠性**和**零知识性**。                                                                                                                                     |
| Sigma 协议                           | Sigma Protocol                                   | 一种特定的三步（承诺、挑战、响应）公共硬币零知识证明协议，用于关系证明。                                                                                                                                                                                                                                           |
| 完备性                               | Completeness                                     | 如果陈述是真实的并且双方都是诚实的，验证者将以概率$1$（或压倒性的高概率）接受证明。                                                                                                                                                                                                                              |
| 可靠性                               | Soundness                                        | 如果陈述是错误的，没有作弊的证明者能够说服诚实的验证者它是真实的，除非概率可忽略。（2-特殊可靠性意味着如果证明者可以回答同一第一消息的两个不同挑战，则可以提取见证）。                                                                                                                                             |
| 零知识性                             | Zero-Knowledge                                   | 验证者除了陈述的真实性之外什么也学不到。（特殊诚实验证者零知识性意味着模拟器可以为诚实验证者生成与真实交互不可区分的记录）。                                                                                                                                                                                       |
| 同态加密 (HE)                        | Homomorphic Encryption (HE)                      | 一种加密方案，允许直接对密文进行计算，使得解密计算结果与对明文执行操作的结果相同。例如，对于加法同态加密：$Enc(m_1) \boxplus Enc(m_2) = Enc(m_1 + m_2)$。                                                                                                                                                        |

### 安全模型与框架 (Security Models & Frameworks)

| 术语 (Term - Chinese)         | 英文 (English)                          | 解释 (Explanation)                                                                                                                                                                                                              |
| :---------------------------- | :-------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 安全模型                      | Security Model                          | 用于定义和分析密码协议安全性的形式化框架，规定了敌手的能力和安全目标。                                                                                                                                                          |
| 威胁模型                      | Threat Model                            | 安全定义的一部分，规定了敌手的能力，例如他们可以看到哪些信息以及可以执行哪些操作。                                                                                                                                              |
| 理想功能 (F)                  | Ideal Functionality                     | 对密码任务或协议预期实现的功能的理想化描述。它通常涉及一个“可信第三方”，完美地执行所需的计算和信息揭示。通过证明真实协议“模拟”其理想功能来证明其安全性。                                                                    |
| 通用可组合框架 (UC Framework) | Universal Composability (UC) Framework  | 一种强大的密码协议安全框架，保证协议即使在与其他任意协议并发运行或在复杂系统中使用时也保持安全。通过证明真实协议的执行与涉及理想功能和模拟器的理想执行不可区分来证明安全性。                                                    |
| 真实世界                      | Real World                              | 实际协议参与方相互交互以及与敌手交互的执行环境。                                                                                                                                                                                |
| 理想世界                      | Ideal World                             | 参与方与理想功能交互，并且敌手（模拟器）试图模仿真实世界敌手交互的执行环境。                                                                                                                                                    |
| 环境 (Z)                      | Environment (Z)                         | `UC` 框架中的一个实体，它向协议参与方提供输入，接收他们的输出，并与敌手交互以试图区分真实世界和理想世界。                                                                                                                     |
| 敌手 (A)                      | Adversary (A)                           | 试图破坏协议安全性的真实世界中的恶意实体。                                                                                                                                                                                      |
| 模拟器 (S)                    | Simulator (S)                           | 理想世界中的一个实体，它与理想功能和环境（通常还有一个真实世界敌手的内部副本）交互，以产生与真实世界执行不可区分的交互。                                                                                                        |
| UC-安全实现                   | UC-Secure Realization                   | 如果对于每个真实世界的敌手$A$，都存在一个理想世界的模拟器 $S$，使得没有环境 $Z$ 能够区分在真实世界中与 $\Pi$ 和 $A$ 交互，以及在理想世界中与 $F$ 和 $S$ 交互，则协议 $\Pi$ `UC` 安全地实现了理想功能 $F$。  |
| 并发组合的安全性              | Concurrent Composition Security         | 协议即使在其多个实例或其他协议并发运行时也保持安全的属性。`UC` 安全意味着这一点。                                                                                                                                             |
| 子协议组合的安全性            | Subprotocol Composition Security        | 如果协议$\Pi$ 使用子协议 $\Pi^*$（它安全地实现了理想功能 $F^*$），并且当使用 $F^*$ 作为黑盒时 $\Pi$ 是安全的，那么当 $F^*$ 被 $\Pi^*$ 替换时 $\Pi$ 仍然安全的属性。`UC` 安全提供了这一点（`UC` 组合定理）。 |
| 独立模型                      | Stand-Alone Model                       | 一种安全模型，其中协议的单个实例被孤立地考虑，没有其他协议的并发执行。它通常是比 `UC` 安全更弱的安全保证。                                                                                                                    |
| 攻陷方式                      | Corruption Types                        | 敌手危害协议参与者的不同方式。                                                                                                                                                                                                  |
| 静态攻陷                      | Static Corruption                       | 在协议执行开始前就固定了被攻陷方的集合。                                                                                                                                                                                        |
| 适应性攻陷                    | Adaptive Corruption                     | 敌手可以在协议执行期间根据观察到的信息动态地选择攻陷参与方。                                                                                                                                                                    |
| 可移动攻陷                    | Mobile Corruption                       | 敌手可以攻陷参与方，也可以“解除攻陷”（恢复其诚实状态），可能在参与方之间移动攻陷。                                                                                                                                            |
| 主动安全                      | Proactive Security                      | 即使面对可移动敌手也成立的安全性，其中参与方可以刷新其状态以减轻长期攻陷的影响。                                                                                                                                                |
| 隐私性                        | Privacy                                 | 协议不泄露参与方私有输入的任何信息，除非这些信息可以从其自身输入和协议输出中推断出来。在模拟范式中，这意味着可以模拟被攻陷方的视图，前提是给定他们的输入和输出。                                                                |
| 恶意安全性                    | Malicious Security                      | 针对可以任意偏离协议规范的敌手的安全性。协议仍应保证诚实参与方的正确性和隐私性，或检测作弊行为。                                                                                                                                |
| 半诚实安全 / 被动安全         | Semi-Honest Security / Passive Security | 针对遵循协议规范但试图通过分析其接收到的消息（其记录或视图）来学习额外信息的敌手的安全性。                                                                                                                                      |

### 协议与技术 (Protocols & Techniques)

| 术语 (Term - Chinese)               | 英文 (English)                              | 解释 (Explanation)                                                                                                                                                                                                                               |
| :---------------------------------- | :------------------------------------------ | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ElGamal 加密方案                    | ElGamal Encryption Scheme                   | 一种基于 Diffie-Hellman 问题的非对称密钥加密算法。加密涉及$c_1 = g^r$ 和 $c_2 = m \cdot \text{pk}^r$。解密是 $m = c_2 / (c_1)^{\text{sk}}$。                                                                                               |
| BGW 协议                            | BGW Protocol                                | 由 Ben-Or, Goldwasser 和 Wigderson 提出的安全多方计算协议，通常基于 Shamir 秘密分享。如果$t < n/2$，它可以实现对计算上无界的半诚实敌手的完美安全；如果 $t < n/3$，则可以实现对恶意敌手的完美安全。                                           |
| GMW 协议                            | GMW Protocol                                | 由 Goldreich, Micali 和 Wigderson 提出的安全多方计算协议。布尔版本使用 1-out-of-4 `OT` 进行与门计算和加法秘密分享（$\mathbb{Z}_2$）。算术版本使用茫然线性计算（`OLE`）进行有限域上的乘法门计算。对最多 $n-1$ 个半诚实敌手是安全的。      |
| 算术电路                            | Arithmetic Circuit                          | 使用有限域上的加法和乘法门表示函数的方法。用于 `MPC` 中计算函数。                                                                                                                                                                              |
| 布尔电路                            | Boolean Circuit                             | 使用与、或、异或、非等布尔门表示函数的方法。用于 `MPC`，特别是姚氏协议。                                                                                                                                                                       |
| 姚氏混淆电路协议 (Yao's GC)         | Yao's Garbled Circuits Protocol (Yao's GC)  | 一种经典的两方计算协议，其中一方（混淆者）“混淆”代表函数的布尔电路，另一方（计算者）使用 `OT` 获取输入标签来计算它。                                                                                                                         |
| 混淆电路                            | Garbled Circuit                             | 布尔电路的加密版本，允许在不泄露中间值或电路结构（超出输出所暗示的范围）的情况下进行计算。                                                                                                                                                       |
| 混淆表                              | Garbled Table                               | 对于混淆电路中的每个门，一个包含密文的表，这些密文加密输出线的混淆标签，对应于输入线的可能混淆标签。                                                                                                                                             |
| 标识置换                            | Point-and-Permute                           | 混淆电路的一种优化，其中每个混淆线标签都有一个关联的“选择位”，用于确定要解密混淆表中的哪个条目，从而避免尝试性解密。                                                                                                                           |
| 混淆表行缩减 (GRR)                  | Garbled Row Reduction (GRR)                 | 减少混淆表中密文数量的技术（如 `GRR3`、`GRR2`）（例如，从4行减少到3行或2行）。                                                                                                                                                               |
| 半门                                | Half-Gates                                  | 混淆电路的一种优化技术，将与门（和其他奇数门）的混淆表大小减少到两个密文，同时与 Free XOR 兼容。                                                                                                                                                 |
| Free XOR                            | Free XOR                                    | 混淆电路的一种优化，通过为每条线定义两个混淆标签之间的全局差异$\Delta$，允许以零密码操作（基本上是免费的）计算异或门。                                                                                                                         |
| BMR 协议                            | BMR Protocol                                | 由 Beaver, Micali 和 Rogaway 提出的多方计算协议，将姚氏混淆电路的思想扩展到多方场景。参与方协作生成混淆电路和输入标签。                                                                                                                          |
| BDOZ 协议                           | BDOZ Protocol                               | 一种针对恶意敌手的 `MPC` 协议（Bendlin, Damgård, Orlandi, Zakarias），通常用于 $n-1$ 个腐败方。使用 Beaver 三元组和秘密份额上的 `MAC` 进行认证。通常依赖（半）同态加密进行预处理阶段。                                                    |
| SPDZ 协议                           | SPDZ Protocol                               | 一种针对恶意敌手的 `MPC` 协议（Damgård, Pastro, Smart, Zakarias），通常用于 $n-1$ 个腐败方。使用 Beaver 三元组和全局 `MAC` 密钥来认证份额和计算，旨在在线阶段比 `BDOZ` 更高效。                                                         |
| 切分选择                            | Cut-and-Choose                              | 一种实现针对恶意敌手安全性的技术，尤其是在混淆电路协议中。混淆者创建混淆电路的多个副本；计算者随机选择一个子集进行“检查”（由混淆者打开以验证正确性），其余的用于计算。                                                                         |
| Beaver 三元组                       | Beaver Triples                              | 预计算的相关随机三元组$([a],[b],[c])$，其中 $a,b$ 是随机的且 $c=ab$，每个参与方持有 $a,b,c$ 的份额。用于高效计算 `MPC` 协议中的乘法门：要计算 $xy$，参与方打开 $d=x-a$ 和 $e=y-b$，然后本地计算 $xy=de+d[b]+e[a]+[c]$ 的份额。 |
| 可验证秘密分享 (VSS)                | Verifiable Secret Sharing (VSS)             | 一种秘密分享方案，其中接收份额的参与方可以验证其份额是一致的，并且对应于一个明确定义的秘密（例如，份额位于正确次数的多项式上），即使分发者是恶意的。                                                                                             |
| Reed-Solomon 码                     | Reed-Solomon Code                           | 一种基于在有限域上求值多项式的纠错码。Shamir 秘密分享与 Reed-Solomon 码密切相关。如果错误数量低于某个阈值，它们可以纠正错误。                                                                                                                    |
| 纠错码                              | Error-Correcting Code                       | 用于检测和纠正数据传输或存储过程中可能发生的错误的代码。在 `MPC` 中，即使某些份额被恶意参与方损坏，它们也可以帮助重建值。                                                                                                                      |
| 认证秘密分享                        | Authenticated Secret Sharing                | 份额附带认证信息（例如 `MAC`）以确保其完整性并防止恶意修改的秘密分享方案。用于像 `BDOZ` 和 `SPDZ` 这样的协议。                                                                                                                             |
| 茫然线性计算 (OLE)                  | Oblivious Linear Evaluation (OLE)           | 一种两方协议，其中发送者有输入$(\alpha,\beta)$（来自域 $F$），接收者有输入 $x \in F$。接收者学习到 $\gamma=\alpha x+\beta$，而发送者对 $x$ 一无所知，接收者对 $\alpha,\beta$ 也学不到更多信息。                                      |
| 零知识明文知识证明                  | Zero-Knowledge Proof of Plaintext Knowledge | 一种 `ZKP`，证明者证明他们知道与给定密文 $c$ 对应的明文 $m$，并且可能证明 $m$ 满足某些属性，而不泄露 $m$。                                                                                                                             |
| PSI (Private Set Intersection)      | Private Set Intersection (PSI)              | 一种安全多方计算任务，其中两个或多个参与方，每个参与方持有一组项目，希望计算这些集合的交集，而不泄露任何关于不在交集中的项目的信息。                                                                                                             |
| PIR (Private Information Retrieval) | Private Information Retrieval (PIR)         | 一种协议，允许用户从持有数据库的服务器检索项目，而服务器不知道检索了哪个项目。                                                                                                                                                                   |
